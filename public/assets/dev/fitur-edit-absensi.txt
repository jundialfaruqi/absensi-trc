Konsep Fitur Edit Absensi

Dua Skenario Utama

Skenario A — Record absensi sudah ada (personel sempat absen, tapi statusnya perlu dikoreksi)
Contoh: personel masuk telat, ternyata ada surat tugas → admin ubah status dari TELAT ke DINAS.

Skenario B — Record absensi tidak ada sama sekali (personel tidak hadir, tidak absen)
Contoh: personel tidak masuk hari ini → di database tidak ada record → admin harus membuat record baru dengan status SAKIT, IZIN, atau ALPHA.
Ini penting karena logika di Livewire-nya berbeda: satu update, satu lagi create.

Perubahan Database yang Dibutuhkan
Tabel absensis — tambah kolom:
edited_by_user_id   bigint nullable   → siapa admin yg edit (FK ke users)
edited_at           timestamp nullable → kapan diedit
alasan_edit         text nullable      → alasan/catatan admin
original_status_masuk  varchar nullable → snapshot status sebelum diedit
original_status_pulang varchar nullable → snapshot status sebelum diedit
nomor_surat         varchar(255) nullable → untuk SAKIT/IZIN
cuti_id             bigint nullable    → FK ke cutis (untuk status CUTI)
keterangan          text nullable      → catatan bebas

Perhatikan cuti_id — Anda sudah punya tabel cutis, jadi ini tinggal disambungkan. Ketika admin memilih status CUTI, sistem harus memaksa admin memilih jenis cuti dari dropdown yang mengambil data dari tabel cutis.

Tiga Kasus Edge yang Perlu Ditangani
Kasus 1 — Personel tidak hadir sama sekali (tidak ada record)
Admin harus bisa membuat record absensi manual. Artinya method di Livewire bukan update, tapi create dengan data minimal: personnel_id, jadwal_id, tanggal, dan status yang dipilih. Jam masuk/pulang boleh dikosongkan atau diisi manual oleh admin.
Kasus 2 — Personel sudah absen masuk, tapi tidak absen pulang
Record ada tapi jam_pulang null. Admin bisa override status_pulang menjadi SAKIT/DINAS/dll tanpa harus mengisi jam_pulang — cukup tambahkan status_pulang dan keterangan.
Kasus 3 — Status sudah HADIR tapi ternyata ada surat sakit menyusul
Record lengkap, tapi admin perlu override status. Di sinilah original_status_masuk penting — simpan status lama sebelum ditimpa, agar ada jejak audit.

Siapa saja yang boleh mengedit absen?
user dengan role super-admin bisa edit semua data absen personel, tapi selain itu dia hanya bisa mengedit personel yang ada di OPD nya. Termasuk juga untuk melihat data absensi juga.

🗂️ Gambaran Umum Fitur
Admin dapat mengedit data absensi personel langsung dari matrix tabel di halaman Monitoring Absensi. Klik pada sebuah cell di matrix akan membuka modal edit (DaisyUI modal), di mana admin bisa:

Ubah jam_masuk dan status_masuk (HADIR / TELAT)
Ubah jam_pulang dan status_pulang (HADIR / PC)
Ubah keterangan / alasan (field baru: keterangan_edit)
Set status khusus: SAKIT, IZIN, atau ALPHA (tidak hadir)
Buat record absensi baru jika personel belum absen (admin input manual)

🗂️ Gambaran Wajib yg tidak boleh dilanggar

1. UI harus 100% konsisten dengan halaman Monitoring Absensi yang sudah ada.
